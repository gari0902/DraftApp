# ドラフトアプリ 仕様書

## 概要

プロ野球のドラフト会議を模した、チームメンバー編成アプリ。
1台のデバイス（スマホ or PC）で操作し、複数チームのメンバードラフトを行う。

## 利用形態

- 1台のデバイスで操作（シングルユーザー操作）
- 認証機能なし
- リアルタイム通信なし

## UIモード

2つのモードから選択可能。

### モード1：シンプルUI

- 各代表者が順番にメンバーを選択していくシンプルな画面
- 指名結果がリスト形式で表示される

### モード2：演出あり（ドラフト会議風）

- プロ野球のドラフト会議のような演出付き
- 指名時にアニメーションエフェクトで結果を表示
- カード反転、スライドインなどの視覚効果

## ドラフトの流れ

### 1. 事前準備

1. チーム数・チーム名を入力（可変）
2. メンバープール（選ばれる側）を入力（可変）
3. UIモード（シンプル / 演出あり）を選択

### 2. 1巡目：くじ引き方式

1. 全チームが**同時に**指名したいメンバーを選択する
2. 競合がないチーム → そのまま確定
3. 競合が発生したチーム → **抽選で1チームが獲得**
4. 抽選に外れたチーム同士で再度指名 → 抽選を繰り返す
5. **全チームが1人確保するまで**繰り返す

### 3. スネークドラフト順の決定

1巡目の結果に基づき、**抽選の結果が悪い順（指名確定が遅かったチーム順）** から先に選べる順番を決定する。

#### 例（4チームの場合）

| チーム | 1巡目の結果 | 確定タイミング |
|---|---|---|
| チームA | 競合なし → 一発確定 | 1番目 |
| チームB | 競合なし → 一発確定 | 1番目 |
| チームC | 競合 → 抽選勝ち | 2番目 |
| チームD | 競合 → 抽選負け → 再指名で確定 | 3番目 |

→ スネーク順: **D → C → A → B**（同率はランダムで決定）

### 4. 2巡目以降：スネークドラフト

決定した順番に従い、蛇行式（スネーク）で指名を行う。

```
2巡目:  D → C → A → B
3巡目:  B → A → C → D（逆順）
4巡目:  D → C → A → B
5巡目:  B → A → C → D（逆順）
  ：
```

- 順番制のため競合は発生しない（先取り）
- 各チームは残りのメンバープールから1人を選択する

### 5. 終了条件

- メンバープール全員が選ばれたら終了
- 割り切れない場合、残りメンバーが足りないチームはスキップ

## 技術構成

| レイヤー | 技術 | 備考 |
|---|---|---|
| フレームワーク | Next.js (App Router) | React内包、Vercel最適 |
| 言語 | TypeScript | 型安全 |
| スタイリング | Tailwind CSS + shadcn/ui | ユーティリティCSS + UIコンポーネント |
| アニメーション | Framer Motion | モード2の演出用 |
| DB | Neon DB (PostgreSQL) | サーバーレスPostgreSQL |
| ORM | Drizzle ORM | 型安全、Neon公式サポート |
| デプロイ | Vercel | Next.jsとの最適な統合 |
| ソース管理 | GitHub | チーム開発 |

## 画面構成（想定）

1. **トップページ** - モード選択、ドラフト作成
2. **セットアップ画面** - チーム名・メンバープール入力
3. **1巡目画面** - くじ引き（同時指名 → 抽選）
4. **ドラフト画面** - スネークドラフト進行
5. **結果画面** - 各チームの最終メンバー一覧
