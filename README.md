# 開発ルール
## 📌 概要

本リポジトリはチーム開発用です。
全員が同じルールで開発を進めるため、以下を遵守してください。

---

## 🧑‍💻 ブランチ運用

基本構成

```
main        本番用
develop     開発用
feature/*   機能開発
```

ルール

* mainへ直接push禁止
* develop経由でマージ
* 作業は必ずfeatureブランチ

例

```
feature/login-page
```

---

## 📝 Issueルール

* 作業前にIssue作成
* Issueに目的・内容を記載
* 完了したらclose

タイトル形式

```
[機能] ログイン画面作成
[修正] ヘッダー崩れ対応
```

---

## 🔀 Pull Requestルール

PR作成時は以下を記載

* 目的
* 変更内容
* 確認方法

チェックリスト

* [ ] ビルド通る
* [ ] Lintエラーなし
* [ ] 不要ログ削除済み

---

## 👀 レビュー

* 最低1人レビュー必須
* 自己マージ禁止
* 指摘は必ず対応 or 理由コメント

---

## 📦 コミットメッセージ規則

形式

```
種別: 内容
```

例

```
feat: ログイン機能追加
fix: バリデーション修正
refactor: API処理整理
```

種別一覧

* feat → 機能追加
* fix → バグ修正
* refactor → リファクタ
* docs → ドキュメント
* style → 見た目のみ
* chore → その他

---

## 🚫 禁止事項

* main直push
* 動かない状態でPR

---

## 🎯 開発方針

* 小さく作る
* 小さくPR出す
* 早めにレビュー

---

## 🆘 トラブル時

分からないことがあれば
**勝手に進めず相談すること**
